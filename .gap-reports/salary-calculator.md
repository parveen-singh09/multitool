# salary-calculator
verdict: gaps

## Competitors
1. calculator.net (salary-calculator) — key features: amount with 8-frequency selector (hour/day/week/bi-week/semi-month/month/quarter/year); hours per week, days per week, holidays per year, vacation days per year; results table across 8 frequencies with TWO columns — Unadjusted and Holidays-&-vacation adjusted; save option.
2. Omni Calculator (finance/salary-to-hourly) — key features: bidirectional fill (enter any one wage, rest populate); hours/week configurable; outputs hourly/daily/weekly/monthly/annual PLUS per-minute and per-second under "other time intervals"; currency auto-set by country.
3. TheCalculatorSite salary calculator — key features: multi-currency salary/pay conversion across standard frequencies (general-purpose converter in the same class).

## Our current features
- Rate + pay-basis selector (hourly/daily/weekly/bi-weekly/monthly/annual)
- Hours per week, days per week, weeks per year inputs
- Outputs annual/hourly/daily/weekly/bi-weekly/monthly
- Fully client-side

## Gaps to close (only genuine missing/worse features; empty if already-wins)
- [ ] Holidays & vacation adjustment (unadjusted vs adjusted) — calculator.net's headline differentiator: subtract paid holidays and vacation days to show adjusted figures alongside raw ones. Add holidays/year and vacation-days/year inputs and a second adjusted column. This is the main gap for a salary tool.
- [ ] Semi-monthly and quarterly outputs — competitor shows 8 frequencies; we show 6 (missing semi-monthly and quarterly). Two extra derived divisions (annual/24, annual/4).
- [ ] Per-minute / per-second breakdown — Omni offers it as a novelty; cheap to add from hourly.
